# Screenshots

This directory contains screenshots demonstrating the implementation and validation of the AWS Threat Detection Pipeline.

## Included Screenshots

### CloudTrail

Shows the multi-region CloudTrail configuration used to collect audit events across the AWS environment.

### GuardDuty

Demonstrates GuardDuty configuration and threat detection capabilities.

### SNS and SQS Integration

Shows the event delivery mechanism used to transfer CloudTrail events from Amazon S3 to Splunk.

### Splunk Ingestion

Demonstrates successful ingestion of AWS CloudTrail events into Splunk using the AWS Add-on and SQS-Based S3 Input.

### Detection Rules

Shows custom Splunk detection rules developed to identify suspicious AWS activity and validate monitoring coverage.

## Purpose

These screenshots provide visual evidence of the deployed infrastructure, log ingestion pipeline, and security monitoring capabilities described in this project.
